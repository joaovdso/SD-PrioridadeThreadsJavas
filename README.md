# SD-PrioridadeThreadsJavas

O código apresenta um exemplo de concorrência em Java com duas threads de prioridades distintas: a classe BaixaPrioridade, que define sua prioridade como mínima e executa um laço infinito imprimindo continuamente uma mensagem, e a classe AltaPrioridade, que define prioridade máxima, imprime cinco vezes uma mensagem em loop infinito e realiza pequenas pausas com sleep(10), encerrando o programa caso seja interrompida durante o sono. Na classe principal LancadorPrioridade, ambas as threads são iniciadas, o método yield() é usado para permitir que o escalonador dê preferência às outras threads, e um shutdown hook é registrado para interromper as duas threads caso o programa seja finalizado externamente. A thread principal então dorme por 50 segundos antes de tentar interromper manualmente ambas as threads e imprimir uma mensagem de encerramento, demonstrando o uso de prioridades de execução, loops infinitos, interrupções e finalização controlada de threads na JVM.
